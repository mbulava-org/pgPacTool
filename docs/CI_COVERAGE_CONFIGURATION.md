## 🎯 Overview

The CI/CD pipeline is configured to measure **real source code coverage** by excluding:
- ✅ Generated protobuf files (~273 classes)
- ✅ Files in obj/Debug directories
- ✅ LinuxContainer.Tests project (local dev only)

This provides an accurate view of how well our actual source code is tested.

---

## 📋 Configuration Files

### 1. `coverlet.runsettings`
Main coverage configuration file used by both CI and local development.

**Exclusions**:
```xml
<Exclude>
  [*]*Protos.*
  [*]*.Protos.*
  [*]*PgQuery.*
  [*.Tests]*
  [*Tests]*
</Exclude>

<ExcludeByFile>
  **/*obj/**/*.cs
  **/Protos/*.cs
  **/*.Designer.cs
  **/*.generated.cs
</ExcludeByFile>
```

### 2. `Directory.Build.props`
MSBuild properties applied to all projects.

**Key Settings**:
- Automatic coverlet.collector reference for test projects
- ExcludeByFile patterns
- ExcludeByAttribute configuration

### 3. GitHub Actions Workflow
`.github/workflows/build-and-test.yml`

**Key Changes**:
```yaml
# Test execution with exclusions
dotnet test \
  --filter "FullyQualifiedName!~LinuxContainer.Tests" \
  --settings coverlet.runsettings

# Report generation with filters
classfilters: '-PgQuery.*;-*.Protos.*'
filefilters: '-**/obj/**;-**/Protos/**'
```

## 🚫 What's Excluded and Why

### LinuxContainer.Tests Project ❌
**Reason**: Designed for local multi-platform development testing
- Tests native library loading in Linux containers
- Validates cross-platform compatibility
- Not needed in CI (runs on Linux already)
- Will have dedicated multi-platform CI workflow later

**How**: Test filter `FullyQualifiedName!~LinuxContainer.Tests`

### Generated Protobuf Code ❌
**Reason**: Auto-generated by protobuf compiler
- ~273 classes (89% of Npgquery codebase)
- ~10,000+ lines of generated code
- Not code we wrote or should test
- Testing generated code provides no value

**How**: Multiple layers
1. RunSettings exclude patterns
2. ReportGenerator class filters
3. ReportGenerator file filters

### Files in obj/Debug ❌
**Reason**: Build artifacts
- Temporary generated files
- Not source code
- Changes on every build

**How**: `ExcludeByFile` pattern `**/*obj/**/*.cs`

## 🚀 Usage

### CI/CD (Automatic)
```yaml
# Runs on every push/PR
# Excludes LinuxContainer.Tests
# Uses coverlet.runsettings
# Reports accurate source coverage
```

### Local Development
```powershell
# Get accurate coverage report
.\scripts\Get-AccurateCoverage.ps1 -Project "Npgquery"

# Run tests with coverage
dotnet test --settings coverlet.runsettings --collect:"XPlat Code Coverage"

# Run tests excluding Linux container tests
dotnet test --filter "FullyQualifiedName!~LinuxContainer.Tests"
```

### Quick Commands
```powershell
# All tests except LinuxContainer, with coverage
dotnet test `
  --filter "FullyQualifiedName!~LinuxContainer.Tests" `
  --collect:"XPlat Code Coverage" `
  --settings coverlet.runsettings

# Npgquery tests only
dotnet test tests\Npgquery.Tests\Npgquery.Tests.csproj `
  --collect:"XPlat Code Coverage" `
  --settings coverlet.runsettings

# mbulava.PostgreSql.Dac tests only
dotnet test tests\mbulava.PostgreSql.Dac.Tests\mbulava.PostgreSql.Dac.Tests.csproj `
  --collect:"XPlat Code Coverage" `
  --settings coverlet.runsettings
```

## 🎓 Best Practices

### When Adding New Features
1. ✅ Write tests first (TDD)
2. ✅ Use `coverlet.runsettings` locally
3. ✅ Aim for 75%+ on new code
4. ✅ Don't worry about generated code

### When Reviewing PRs
1. ✅ Check coverage report in PR comment
2. ✅ Verify new features have tests
3. ✅ Ensure coverage doesn't drop significantly
4. ✅ Remember threshold is source-only (50%)

### When Coverage Drops
1. Identify which source files lost coverage
2. Add targeted tests
3. Don't add tests for generated code
4. Focus on critical paths
